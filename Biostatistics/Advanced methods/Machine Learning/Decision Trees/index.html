Decision trees

Background

  • developed from a series of questions (generally binary yes/no questions)

  • Form the basis of Ensemble learning: Random Forests and Bagging and pasting and Boosting

    • Bagging - idea is similar to bootstrapping, where multiple trees are built using subsets of data with replacement
    • Random forests - multiple trees are built using subsets of the features
    • Boosting - place different weights on different samples in each iteration (resulting in different shaped trees)
  • Benefits of the decision tree
    • Requires very little data preparation (including feature scaling or centreing)
    • Each node can be split
      • into binary outcomes: using the CART algorithm
      • into >2 children: using ID3 algorithm (and others)
    • prediction only takes O(log2(m)) - it is very fast to predict even with large training sets
    • non-parametric model
    • highly explainable model
  • Disadvantages
    • Finding the optimal tree is an NP-Complete problem (i.e. a very hard problem to solve)
    • this algorithm is very expensive to train - O(n * m * log(m))
    • being nonparametric it is at risk of overfitting
    • Dscision trees are unstable
      • orthogonal decision boundaries (i.e. x1 < 1 | x1 >= 1) due to decisions made on a single variable
        • if the decision boundary is anything but horizontal or vertical, the split will suffer
        • this can be improved using PCA (Which can reorient the data)
      • very dependent on training data
      • training algorithm may be stochastic (as in Scikit-learn) - i.e. the feature selected to evaluate at each node is random - this can result in different models trained even on the same training data
        • this can be minimised by averaging predictions over many trees
      • reaches homogonous nodes (or runs out of attributes) so datasets will continue to split nodes (unless regularised) until the very end → heavily contributes to overfitting
        • e.g. predicting gender; given height, weight, last book read, colour of rice eaten
          • the last 2 questions will be irrelevant to classification however given we have no further questions to ask, they will be used if they can split the data at all

Use cases

  • Classification and regression (and multioutput)

Pseudocode

Function BuildTree(n, A) // n: samples (rows), A: attributes
    If empty(A) or all n(L) are the same
        status = leaf
        class = most common class in n(L)
    else
        status = internal
        a <= bestAttribute(n, A)
        LeftNode = BuildTree(n(a=1), A \ {a}) // remove the attribute {a} after this
        RightNode = BuildTree(n(a=0), A \ {a})
    end
end

Mathematics

Classification and Regression Tree (CART) algorithm

  1. Algorithm splits training set into two subsets using a single feature \(k\) and threshold \(t_k\)
    • \(k\) and \(t_k\) are determined by finding the pair which produces the purest subsets
  2. The algorithm recursively splits the subsets using the same logic
  3. it will stop when it reaches the maximum depth or if it cannot find a split which reduces impurity

Issues with CART

  • CART is a greedy algorithm and is not guaranteed to produce an optimal solution

CART cost function

\[ J(k, t_k) = \frac{m_{left}}{m}\times G_{left} + \frac{m_{right}}{m} \times G_{right} \]

Where:

  • \(G\) measures the impurity of the left/right subset in classification trees
    • in a regression trees, G is replaced with the Mean squared error (MSE)
  • \(m\) measures the number of instances in the left/right subset

Hyperparameters

  • Increase to regularise:
    • Minimum depth of the Decision Tree
    • Minimum number of samples a node must have before it can be split (to prevent a large number of leaf nodes)
    • Minimum number of samples a leaf node must have
  • Decrease to regularise:
    • Maximum number of leaf nodes
    • Maximum features evaluated for splitting at each node

Pruning algorithm

  • Decision trees are trained without restrictions
  • effectively removing bad questions out of the tree
  • The tree is then pruned (deleted) according to a statistical test (such as the \(\chi^2\) test) with a threshold (e.g. p < 0.05)
    • if the purity improvement is not statistically significant due to the node, the node and its leafs are deleted
  • Procedure
    • start with test set
    • starting at the root/internal nodes
    • calculate the test set initial error
      • remove subtree rooted at node
      • assign class to be the most common among training set
      • check test data error
        • if error is lower, keep change
        • otherwise restore subtree, repeat for all nodes in subtree
  • Rough pruning methods
    • set a max tree length
    • typically used when combining decision trees (E.g. used in boosting)

Analysis

  • Impurity / best attribute (for classification tree)
    • Gini impurity
      • faster algorithm
    • Entropy and Information gain
      • more balanced trees
  • Mean squared error (MSE) (for regression trees)
  • FFeature importance in tree based classifiers

Intuition on model fitting

  • A tree consists of internal nodes corresponding to attributes (Features)
    • can have n layers/questions
  • it also consists of leafs (a.k.a. terminal nodes) - corresponding to classification outcomes
  • edges denote assignments of each node

Explanations

Model prediction

Suppose you want to classify an item given some set of features which was used to fit the decision tree model.

  • Starting at the first split (i.e. the root node), you determine whether the feature of interest in this root node results in you moving to whichever child
  • at each child node you repeat this process
  • this occurs until you hit a leaf node (i.e. one with no further children)

Code

Classification tree

from sklearn.datasets import  load_iris
from sklearn.tree import DecisionTreeClassifier

iris = load_iris()
X = iris.data[:, 2:] # petal length and width
y = iris.target

tree_clf = DecisionTreeClassifier(max_depth=2)
tree_clf.fit(X, y)

from sklearn.tree import export_graphviz

export_graphviz(
    tree_clf,
    out_file = "iris_tree.dot",
    feature_names = iris.feature_names[2:],
    class_names = iris.target_names,
    rounded=True,
    filled=True
)

Regression tree

from sklearn.tree import DecisionTreeRegressor

tree_Reg = DecisionTreeRegressor(max_depth = 2)
tree_reg.fit(X, y)
